# 游戏设计文档：《墨缚之塔》

> **文档状态：** v0.1.0 规则重构稿
> **适用引擎：** Godot 4.x
> **设计基线：** 本文档全面取代旧版 v0.1.0 战斗规则。被替代内容见第 13 章“已废止规则记录”，不得继续作为实现依据。

## 1. 游戏概览

- **标题：** 墨缚之塔
- **类型：** 肉鸽卡组构筑 + 交替出牌自动战斗 + 六边形区域控制策略
- **视角：** 2D 俯视略微倾斜，可滚动、缩放的大型六边形卷轴战场
- **平台：** Godot 4.x，桌面端，鼠标 + 键盘
- **一句话卖点：** 进入法师之塔，通过卡牌、墨水经济与召唤的书灵，在充满活力的六边形卷轴战场上争夺控制权。
- **目标单次游玩时长：** 完整肉鸽流程 30-45 分钟；v0.1.0 为简短闭环：选择房间 -> 战斗 -> 奖励 -> 下一房间或退出。

玩家不直接控制任何战场单位。玩家每个行动回合只能进行一次有效操作，通过卡牌、角色技能、牺牲手牌或结束回合来影响战局。操作完成后，己方全部书灵按照出生顺序自主行动。战斗的核心不是消灭敌方角色，而是在固定边界的六边形卷轴上持续涂色、争夺墨井并取得超过 80% 的区域控制。

### 设计支柱

1. **一次决策，一次自动演算：** 每次出牌、使用技能或完成一次最多两张牌的牺牲流程后，立即结束当前方的操作阶段。
2. **间接控制：** 玩家构筑的是规则、节奏与倾向，不直接下达书灵移动指令。
3. **区域胜于击杀：** 攻击用于清除阻碍和保护扩张，胜负最终由涂色面积决定。
4. **可预测的自动化：** 书灵 AI、敌人三回合意图和狂热优先级都必须可观察、可复现、可测试。
5. **角色机制改变地图：** 茜娜围绕飞行与降落，菲涅围绕精研、护盾与结晶化。

## 2. 核心游戏循环

### 2.1 即时循环

1. 阅读双方涂色比例、墨井归属、书灵位置、敌人未来三次意图与当前手牌。
2. 从“打出一张牌、使用角色技能、进入最多两张牌的牺牲流程、直接结束回合”中选择一项。
3. 有效操作立即锁定，并进入己方运算阶段。
4. 己方书灵按出生顺序依次执行移动、涂色、攻击判定和触发效果。
5. 执行回合结束结算，包括浸润、墨井检查、持续效果推进和胜负检查。
6. 切换行动方；敌方遵守完全对称的基础规则。

### 2.2 战斗循环

```text
行动方回合开始
  -> 墨水与持续效果更新
  -> 单次操作窗口
  -> 有效操作锁定并立即结束操作窗口
  -> 己方书灵按出生顺序运算
  -> 浸润与墨井结算
  -> 在完整双方行动周期结束点检查 >80% 胜利
  -> 狂热/第 90 计数点检查
  -> 若战斗未结束则切换行动方
```

### 2.3 房间与单局循环

```text
主菜单
  -> 角色选择
  -> 卡组构筑、重排与狂热优先级设置
  -> 简易房间地图与本次对手选择
  -> 战斗
  -> 战斗结果
  -> 三选一奖励
  -> 简易房间地图
  -> 下一房间或退出
```

v0.1.0 仅要求一个可重复进入战斗的简易房间地图。完整楼层、事件、商店、精英与首领结构属于后续版本。

### 2.4 跨局成长

长期版本可解锁角色卡牌、低稀有度卡牌、起始卡组变体、角色外观和构筑预设。跨局成长不得提供永久数值碾压，目标是增加构筑可能性与角色玩法差异。

## 3. 核心机制

### 3.1 术语与计数单位

- **行动回合（Turn）：** 单个阵营从回合开始到回合结束的完整过程。
- **完整轮（Round）：** 先手方与后手方各完成一个行动回合。
- **全局行动序号：** 每完成一个行动回合加 1。v0.1.0 固定玩家先手、敌人后手，因此偶数序号结束点同时是完整轮结束点。
- **出生顺序：** 书灵进入战场时获得的单调递增序号。运算阶段按该序号从小到大执行。
- **有效操作：** 成功打出卡牌、成功使用技能、成功牺牲手牌或主动结束回合。
- **无效操作：** 目标非法、费用不足、召唤位置非法或效果足迹越界。无效操作不消耗卡牌、技能、墨水或行动回合。

- **阈值计数：** 50、70、90 均按已完成的全局行动回合数计算。完成第 50 个行动回合后，下一回合进入狂热；完成第 70 个行动回合后，下一回合进入 3x 加速；完成第 90 个行动回合后，在 `>80%` 判定之后执行最终格数结算。
- 所有阈值仍必须集中保存在 `BattleRules` 中，不得散落为硬编码常量。

### 3.2 交替出牌状态机

#### 回合开始阶段

1. 设置当前行动方。
2. 结算该方“回合开始”墨水产出和角色/卡牌持续效果。
3. 该方角色技能冷却推进 1。
4. 确保手牌不超过 5 张；需要补牌时从牌堆抽取。
5. 若为敌方回合，读取并锁定当前要执行的已公开意图。

#### 单次操作阶段

行动方只能完成以下一项：

| 操作 | 前置条件 | 结果 |
|------|----------|------|
| 打出一张书灵卡 | 墨水足够，召唤位置有效 | 扣除墨水、召唤书灵、卡牌进入弃牌堆、立即补牌、结束操作阶段 |
| 打出一张效果卡 | 墨水足够，目标与范围有效 | 扣除墨水、效果入队并完整结算、卡牌进入弃牌堆、立即补牌、结束操作阶段 |
| 使用角色技能 | 技能冷却为 0，目标有效 | 技能生效并设置冷却，结束操作阶段 |
| 牺牲手牌 | 手牌存在且可牺牲 | 第一张进入弃牌堆、回复 0.5 墨水并立即补牌；随后只能再牺牲一张或结束回合 |
| 直接结束回合 | 无 | 不出牌、不牺牲，直接进入运算阶段 |

除牺牲子流程和召唤失败等明确例外外，任何有效操作都立即结束当前方的操作阶段，不能在同一行动回合连续打出多张牌。

牺牲子流程规则：

1. 每个行动回合最多牺牲 2 张牌。
2. 牺牲第一张后立即获得 0.5 墨水并补牌至 5 张。
3. 第一张牺牲后，操作界面只保留“再牺牲一张”和“结束回合”两个选项。
4. 牺牲第二张后再获得 0.5 墨水并补牌，操作阶段自动结束。
5. 因此一次完整双牺牲总计获得 1 墨水。

#### 运算阶段

当前方所有存活书灵按出生顺序依次执行：

1. 选择 AI 目标。
2. 计算最多不超过移动力的路径。
3. 逐格移动；每一步都检查边界、阻挡和特殊状态。
4. 按涂色速率修改本步涉及的格子。
5. 检查攻击目标；发生攻击时双方同时造成各自攻击力伤害。
6. 处理耐久归零、护盾消耗、死亡移除和由移动/涂色/攻击引发的效果。
7. 完成该书灵行动后再处理下一个出生序号。

死亡书灵不得继续执行已排队的移动、涂色、攻击或触发。

#### 回合结束阶段

1. 结算当前方“回合结束”效果。
2. 对全图执行浸润判定。
3. 对三个墨井执行包围与归属判定。
4. 更新玩家、敌人和中立格数量及百分比。
5. 若当前行动方为后手方，则在下一完整轮开始前执行严格大于 80% 的胜利判定。
6. 将已完成的全局行动回合数加 1；若后手方结束，则完整轮数加 1。
7. 在全局行动数达到 50 时标记下一回合进入狂热；达到 70 时标记下一回合进入 3x 加速。
8. 若全局行动数达到 90，在 `>80%` 判定之后执行最终格数结算。
9. 若战斗未结束，切换行动方。

### 3.3 墨水经济

- 双方初始当前墨水为 **20**。
- 双方初始墨水上限为 **20**。
- 每完成 2 个行动回合（包含玩家和敌人回合，即通常一个完整轮），双方墨水上限同时 +1。
- 回合开始时保留当前剩余墨水，**不会自然回复**，也不会自动补满至上限。
- 每个己方已占领墨井在该方回合开始时提供 **1 墨水**。
- 牺牲一张手牌立即回复 **0.5** 墨水，但仍受墨水上限约束。
- 每个行动回合最多牺牲两张牌；完整双牺牲提供 1 墨水并自动结束操作阶段。
- 卡牌效果和书灵效果可按各自文本提供墨水。
- UI 使用整数数字加半格墨水图标显示 0.5；逻辑层使用半单位整数存储，推荐 `ink_half_units: int`，例如 20 墨水存为 40，避免浮点误差。
- 墨水不足时，卡牌保持可查看但不可打出。

### 3.4 手牌、牌堆与弃牌堆

- 手牌上限固定为 5。
- 成功打出或牺牲一张牌后，该牌进入弃牌堆，并立即抽牌补至 5 张。
- 牌堆为空且需要抽牌时，立即将弃牌堆洗回牌堆后继续抽取，不打断当前结算。
- 若牌堆和弃牌堆均为空，手牌可少于 5 张。
- 卡牌实例必须保存稳定 ID、来源角色、费用、类型、目标规则、狂热优先级和当前手牌排序。
- 卡组构筑界面支持拖拽重排；重排顺序同时作为同优先级卡牌的自动出牌顺序。

### 3.5 六边形地图构建

#### 尺寸与拓扑

- 地图名义尺寸为 **60 列 x 30 行**，固定边界，不循环、不包裹。
- 行长度交替：
  - 第 1、3、5……奇数行：60 格。
  - 第 2、4、6……偶数行：61 格。
- 按上述规则，实际可涂色格总数为 **1815**。所有占领百分比以实际存在的格子总数为分母。
- 偶数行相对奇数行偏移半个六边形尺寸，遵循项目既有 odd-r HexMap 布局。用户指定的偏移表达为 `size.y * 0.5`；具体应用于屏幕 x/y 轴的方式必须与 `battle_scroll_field` 视觉和坐标往返测试一致。
- 逻辑坐标使用 Axial 或 Cube 坐标。渲染层可保留 odd-r 行列坐标，但邻居、距离、范围、路径和对称映射必须通过统一坐标工具转换。
- 所有格子必须有稳定坐标键，不得通过场景节点路径作为逻辑 ID。

#### 边界规则

- 任意目标坐标在地图外时，目标无效。
- 所有效果禁止越界，不允许环绕、镜像或自动夹紧到边缘。
- 召唤目标越界或被占用时，召唤失败。
- 移动的下一步越界时，该路径在边缘终止，不生成边界外涂色或攻击。
- 需要完整范围的技能若范围足迹越界，默认整个目标选择无效；该默认解释列入第 12 章确认项。

#### 初始状态

- 所有格子的 `owner` 初始为 0，即中立。
- 占领状态：
  - `0`：中立，无阵营墨水特效或仅显示浅色纸面。
  - `1`：玩家，显示深蓝/黑色墨迹。
  - `2`：敌人，显示红色/紫罗兰墨迹。
- 每个格子还包含可扩展属性槽，用于结晶化、精研、临时禁区、法术免疫等角色或卡牌状态。
- 格子状态不得用互斥布尔值堆叠；推荐使用基础归属 + 标记集合 + 来源/持续时间数据。

#### 高度与占用槽

每个逻辑六边形坐标包含两个独立占用槽，使用 `(occupied, height)` 复合状态：

| 状态 | 含义 |
|------|------|
| `(0, 1)` | 地面无占据 |
| `(1, 1)` | 地面有占据 |
| `(0, 2)` | 空中无占据 |
| `(1, 2)` | 空中有占据 |

- 高度 `1` 为地面，高度 `2` 为空中。
- 同一坐标的地面槽最多存在一个书灵，空中槽最多存在一个书灵。
- 因此同一六边形可同时存在一个地面书灵和一个空中书灵，但不能存在两个同高度书灵。
- 书灵保存自身 `height`。地面书灵只能移动到 `(0, 1)`，空中书灵只能移动到 `(0, 2)`。
- 墨井占用地面槽，状态为 `(1, 1)`；地面书灵不能进入墨井坐标，空中槽仍可独立使用。
- 地图归属、结晶化和精研属于六边形格本身，不随高度层重复保存。

#### 视觉布局

- 六边形网格显示在方形卷轴内部。
- 卷轴整体左右对称，具有轻微近大远小的透视感，风格与 `assets/ui/battle_scroll_field.png` 一致。
- 每格显示低对比度六边形轮廓。
- 归属主要通过墨迹填充、边界晕染和阵营色块表达，网格线只承担定位作用。
- 逻辑坐标与视觉透视必须分离：点击拾取先逆变换到未透视的局部地图空间，再进行 hex 坐标判断。

### 3.6 涂色规则

旧版 Direct Claim、Channel Claim、Fortify、Freeze 占领协议全部废止。统一改为书灵的 **涂色速率**：

| 涂色速率 | 每次移动步骤的涂色结果 |
|----------|------------------------|
| 0 | 不产生被动涂色 |
| 1 | 将书灵脚下当前格改为所属方 |
| 2 | 修改当前格，并额外修改相对移动方向左侧的一格 |
| 3 | 修改当前格，并额外修改相对移动方向左右两侧各一格 |

执行要求：

1. 涂色只修改地图内存在的格子。
2. 左/右侧格必须根据“本次移动方向”在六边形六方向中的朝向计算，不得使用屏幕左右近似。
3. 没有发生位移但需要涂色时，仅处理脚下格；速度 2/3 的侧向格是否处理见第 12 章。
4. 飞行状态中的书灵失去被动涂色能力，视为本次行动涂色速率 0。
5. 结晶地块无法被任何地块效果影响：不能改变归属，不能被重置为中立，也不能新增、移除或覆盖其他地块状态。书灵仍可在其上移动、战斗并成为效果目标。
6. 卡牌明确写有“重置为白色”时，在新版三态归属中解释为重置为中立 `owner = 0`。

### 3.7 浸润

每个行动回合结束时，在书灵行动和普通涂色完成后执行一次全图浸润：

1. 基于浸润开始前的地图快照统计相邻格，避免扫描顺序造成连锁差异。
2. 若某格至少有 5 个有效相邻格属于玩家，则该格变为玩家归属。
3. 若某格至少有 5 个有效相邻格属于敌人，则该格变为敌人归属。
4. 地图边缘不足 6 个邻居的格子仍按“至少 5 个实际相邻格同方”判定。
5. 结晶化或明确禁止涂色的格子跳过浸润。
6. 若同一次快照出现双方都满足条件的理论冲突，则保持原归属并记录规则告警；正常六邻接下不应出现该情况。

### 3.8 墨井

#### 生成

- 地图按列范围在后台划分为左 33%、中 33%、右 33% 三个逻辑区域，不显示区域边界。
- 每个区域随机放置 1 个墨井，共 3 个。
- 墨井锚点距离地图边缘至少 4 格。
- 生成必须使用可记录的随机种子，使战斗可重放、可测试。
- 三个墨井不得占用同一格，也不得落在非法或预留出生格。

#### 归属

- 墨井初始中立。
- 墨井只有在其所有有效周边格均属于同一方时才被该方包围。
- 回合结束时，在浸润之后检查包围。
- 墨井使用以下两步状态转换：
  - 中立墨井被同一方完全包围：变为该方所有。
  - 已占领墨井被对方完全包围：先变为中立。
  - 若下一次检查仍被同一对方完全包围：再变为对方所有。
- 墨井归属独立于锚点格的普通 `owner`，必须单独保存。

#### 收益

- 已占领墨井在占领方每个行动回合开始时提供 1 墨水。
- 收益必须通过墨井组件或数据资源配置，不得写死在 UI。
- 多口墨井收益叠加，但总墨水仍不能超过当前上限。

### 3.9 书灵系统

#### 通用属性

| 字段 | 含义 |
|------|------|
| `attack` | 攻击力；攻击时对目标造成的伤害 |
| `durability` / `max_durability` | 当前与最大耐久度 |
| `movement` | 每次运算阶段最多移动格数 |
| `paint_rate` | 被动涂色速率 0-3 |
| `ai_type` | 避战、扩张或好战 |
| `shield_layers` | 护盾层数；每层抵挡一次伤害事件 |
| `owner` | 玩家或敌人 |
| `birth_order` | 出生顺序 |
| `hex_position` | 当前逻辑坐标 |
| `height` | 1 为地面，2 为空中 |
| `status_effects` | 飞行、精研影响、禁行等状态 |
| `remaining_duration` | 有时限书灵的剩余持续时间 |

书灵数量无上限。同一六边形坐标允许一个地面书灵和一个空中书灵共存，但同一高度层只能存在一个书灵。

#### 初始书灵

双方开局拥有完全相同的三个基础书灵：

| 书灵 | 攻击 | 移动力 | 耐久 | 涂色速率 | AI | 基础目标倾向 |
|------|------|--------|------|------------|----|--------------|
| Spreader | 1 | 2 | 3 | 2 | 避战 | 保持安全距离，优先接近中立墨井并扩展中立区域 |
| Resource | 1 | 2 | 4 | 1 | 扩张 | 贪婪最大化新增涂色，并帮助包围或维持墨井 |
| Fighter | 3 | 2 | 2 | 1 | 好战 | 优先接近可攻击的敌方书灵，其次进入敌方区域 |

- 出生顺序固定为 `Spreader -> Resource -> Fighter`。
- 初始站位为朝向敌人的三角阵型，Fighter 位于最前方。
- 玩家与敌人的阵型相互镜像。
- 精确出生锚点与三角偏移必须作为地图配置数据，见第 12 章。

#### AI 共同目标

所有 AI 使用字典序评分；只有前一项完全相同时才比较后一项：

- **避战型，Spreader 为主：**
  1. 安全性：距离敌方书灵和敌方区域越远越好，避免进入可被攻击的危险格。
  2. 中立墨井接近度：优先接近中立墨井，尤其是己方即将完成包围的墨井。
  3. 本次移动新增涂色数量。
  4. 路径长度，较短者优先。
- **扩张型，Resource 为主：**
  1. 本次移动新增涂色数量。
  2. 墨井包围贡献，包括帮助包围中立墨井或维持己方墨井。
  3. 路径长度，较短者优先。
- **好战型，Fighter 为主：**
  1. 是否存在移动后可攻击的敌方书灵。
  2. 击杀可能，包括本回合击杀和下回合可完成击杀。
  3. 敌方区域深入度。
  4. 本次移动新增涂色数量。
- **完全平局：** 按地图坐标从左到右、从上到下选择。

相同战场状态、随机种子和出生顺序必须产生相同决策。

#### 移动

- 每个书灵每次运算最多移动 `movement` 格。
- 路径逐步执行，允许因阻挡、边缘、死亡或触发效果提前停止。
- 地面书灵只能进入地面无占据槽 `(0, 1)`。
- 空中书灵只能进入空中无占据槽 `(0, 2)`。
- 地面和空中路径分别判定占据；另一个高度层的书灵不会占用当前高度层。
- 墨井的地面槽为 `(1, 1)`，阻止地面书灵进入。
- 飞行书灵可跨越地面障碍与地面单位，但每个实际停留坐标的空中槽必须有效。
- 每一步移动后立即执行该步涂色，再检查由移动触发的卡牌/角色效果。

#### 攻击

- 书灵先完成移动，再从相邻六边形中选择一个可攻击的敌方书灵。
- 普通攻击只能选择高度为 1 的地面目标；攻击者自身可以位于地面或空中。
- 攻击行为类似《炉石传说》的随从交战：攻击方与防守方同时对彼此造成各自攻击力伤害。
- 护盾优先抵挡整个伤害事件并消耗 1 层；被抵挡的一方本次耐久不减少。
- 双方伤害结算后，耐久小于等于 0 的书灵同时标记死亡并移除。
- 死亡书灵不得触发之后的移动或攻击，但可触发明确写明的死亡效果。
- 飞行状态免疫所有攻击，因此不能成为普通攻击目标。
- 同高度与跨高度的其他攻击关系必须由统一战斗规则定义，不得由各书灵场景脚本自行决定。

#### 召唤失败

- 玩家书灵卡召唤位置无效时，卡牌弹回手牌；不扣墨水、不进入弃牌堆、不结束回合，玩家可重新选择。
- 敌方书灵卡召唤失败时，该卡直接牺牲并回复 0.5 墨水；随后敌方再牺牲一张手牌、再回复 0.5 墨水并结束操作阶段。

### 3.10 狂热模式

- 完成第 50 个全局行动回合后，下一回合进入狂热，玩家失去手动操作权，由自动出牌逻辑强制接管。
- 狂热期间书灵继续使用完整 AI，不降低决策质量或省略涂色、攻击、浸润、墨井结算。

#### 自动出牌优先级

1. 玩家可在卡组构筑界面为每张卡设置 `0-9` 数字优先级。
2. `0` 为最高出牌优先级，`9` 为最低出牌优先级。
3. 自动系统按 `0 -> 9` 检查可执行卡牌。
4. 同数字优先级按玩家在卡组构筑界面的卡牌排序从前到后执行。
5. 只有墨水费用足够且存在合法目标/召唤位置的卡才算可打出。
6. 若墨水不足或不存在任何合法可打出的卡，只从红色墨滴标记的可牺牲卡中选择牺牲目标。
7. 牺牲顺序按优先级逆序 `9 -> 0`；同数字时按构筑排序从后到前选择。
8. 灰色墨滴卡永远不能被自动牺牲，无论其优先级数字是多少。
9. 第一张牺牲后，即使新抽到的牌可以打出，也不得改为出牌；狂热仍只能再牺牲一张或结束回合。
10. 若选择继续牺牲，则按同一 `9 -> 0` 规则选择第二张；第二张后自动结束操作阶段。
11. 若手牌中没有红色可牺牲卡，则直接结束回合。
12. 敌方使用其角色/牌组配置的确定性数字优先级与可牺牲状态。

#### 加速

| 阶段 | 自动出牌等待 | 动画速度 | 音频 |
|------|--------------|----------|------|
| 狂热前 | 手动操作 | 1x | 完整音效 |
| 完成全局行动 50 后 | 1 秒/张 | 2x | 仅保留短音效 |
| 完成全局行动 70 后 | 0.5 秒/张 | 3x | 仅保留短音效 |

加速只能改变表现时间，不得改变逻辑步骤、AI 结果、出生顺序或随机数消费顺序。

### 3.11 胜负条件

- 玩家格或敌人格数量 **严格大于** 全部有效格子的 80% 时，该方胜利。
- 按 1815 个有效格计算，最低胜利格数为 **1453**。
- 即时胜利判定只在后手方行动回合结束、下一完整轮开始前进行。
- 完成第 90 个全局行动回合后，在该回合的 `>80%` 检查之后比较双方涂色格数量：
  - 玩家更多：玩家胜利。
  - 敌人更多：敌人胜利。
  - 完全相同：玩家胜利。
- 中立格不计入任何一方。
- 墨井归属不直接增加占领格数，只通过持续墨水和周围地图控制间接影响胜负。
- 结晶化格仍按其基础归属计入占领数量。

### 3.12 卡牌交互

#### 卡牌类型

- **书灵卡：** 在合法格召唤一个书灵。
- **效果卡：** 法术或资源效果，按目标规则立即执行或创建持续状态。

#### 通用卡牌面板

卡牌整体采用常见《游戏王》式竖向集换式卡牌结构：明确的外框、顶部费用/自动设置区、大幅卡图、卡图下方名称与规则文本区。只借鉴信息层级和阅读方式，不复制具体边框、美术或品牌元素。

从上到下的固定布局：

1. **左上角，墨水费用：**
   - 显示消耗墨水数。
   - 使用墨水瓶或墨滴底形承载数字。
   - 半点费用使用整数加半格墨水符号，不显示小数长文本。
2. **右上角，可牺牲墨滴：**
   - 使用红色墨滴形状作为牺牲权限标志。
   - 红色：该卡允许被玩家手动或狂热自动逻辑牺牲。
   - 灰色：该卡不可牺牲，视为 `protected_from_sacrifice = true`。
   - 在数字优先级设置界面中，左键点击墨滴可在红色和灰色之间切换。
   - 鼠标悬浮时显示浮动提示：`可牺牲` 或 `不可牺牲`；不可牺牲时可追加原因，例如“已设为保护卡”。
   - 在战斗手牌中只显示当前状态，不直接切换，避免误触改变构筑设置。
3. **墨滴下方，优先级数字框：**
   - 显示 `0-9` 的单个数字。
   - `0` 为最高自动出牌优先级，`9` 为最低自动出牌优先级。
   - 在构筑/数字优先级设置界面中，玩家可直接点击卡牌上的数字框，打开紧凑的 `0-9` 选择器并立即修改该卡优先级。
   - 鼠标滚轮或键盘方向键可在数字框聚焦时调整数值，但不得超出 `0-9`。
   - 鼠标悬浮显示提示，例如 `自动优先级：0（最高）`。
   - 在战斗手牌中数字框只读，用于解释狂热的出牌和牺牲顺序。
4. **顶部设置区下方，卡图：**
   - 卡图是卡牌面积最大的视觉区。
   - 书灵卡展示书灵主体与角色主题；效果卡展示施法动作或结果。
5. **卡图下方，唯一卡牌名称：**
   - 位于整张卡牌中心偏下。
   - 这是卡牌面板中唯一显示卡牌名称的位置。
   - 长名称允许换成两行，但不得挤压卡图、效果区或右上自动设置区。
6. **名称下方，效果详述：**
   - 显示完整规则文本、数值、目标、持续时间和特殊条件。
   - 战斗手牌缩略态允许裁切为摘要；悬浮放大和详情页必须显示全文。
7. **左下角，扩展字段区：**
   - 放置关键词或角色字段，例如“降落”“飞行”“精研”“结晶化”“ACE”。
   - 字段使用统一图标 + 短标签，并保留可扩展的多行槽位。
   - 新增字段不得改变费用、名称、卡图和效果区的固定位置。

卡牌需要预留以下表现状态：普通、悬浮、拖拽、可打出发光、目标合法、目标非法、费用不足、可牺牲、不可牺牲、优先级数字聚焦、详情放大和狂热选中。

#### 鼠标操作

交互目标是《杀戮尖塔 2》式拖拽出牌：

- 左键按住卡牌：卡牌从手牌槽抬起并悬浮，保留原手牌占位。
- 无目标卡：向上拖拽超过卡牌高度的 50% 后进入发光可释放状态；松开左键打出。
- 有目标卡：向上拖拽超过卡牌高度的 25% 后，卡牌停留在原地悬浮；从卡牌中心绘制矢量箭头到鼠标位置。
- 箭头经过合法目标时，高亮目标并显示可释放状态。
- 松开左键时，若目标合法则打出；否则卡牌回到手牌。
- 右键：
  - 拖拽中：取消拖拽。
  - 非拖拽中：打开卡牌详情。
- 卡牌详情包含完整介绍、规则关键词、来源和“牺牲”按钮。
- 点击“牺牲”后，该牌进入弃牌堆、回复 0.5 墨水并补牌。
- 若这是本回合第一张被牺牲的牌，详情与手牌界面只保留“再牺牲一张”和“结束回合”。
- 若这是第二张被牺牲的牌，操作阶段自动结束。

### 3.13 角色与技能

#### 茜娜

- **主题：** 天鹅、鸟类、飞行、降落、高机动与爆发。
- **角色技能：起风**
  - 冷却：1 个我方行动回合。
  - 使用技能代替本回合出牌并立即结束操作阶段。
  - 选择一个己方书灵，使其进入飞行状态。
  - 目标从地面高度 `1` 转为空中高度 `2`；目标坐标的空中槽必须为 `(0, 2)`。
  - 飞行持续时间等于该书灵的卡牌费用。
  - 飞行期间免疫所有攻击并可跨越障碍。
  - 飞行期间移动不产生被动涂色。
  - 从飞行转为地面时，若当前坐标地面槽不是 `(0, 1)`，移动到最近的合法地面槽；等距离时按从左到右、从上到下选择。
  - 确定合法落点后转为高度 1，并触发“降落”关键词。

#### 菲涅

- **主题：** 结晶化、防御、精研、资源控制。
- **角色技能：精读**
  - 冷却：2 个我方行动回合。
  - 使用技能代替本回合出牌并立即结束操作阶段。
  - 选择己方控制区内一个完整的半径 1 六边形区域，即中心格加六个邻格，共 7 格。
  - 区域内所有己方书灵获得 1 层奥术护盾。
  - 该区域立即触发一次精研判定。

#### 20735

- **主题：** 机械、归档、稳定资源与无人机协同。
- **角色技能：归档**
  - 无冷却；使用技能代替本回合出牌并立即结束操作阶段。
  - 选择一张未归档手牌，等待回合数等于其基础费用。
  - 归档牌保留在手牌并锁定；每个所属方回合结束时等待回合数 -1，归零后免费自动使用。
  - 自动使用由统一目标系统选择合法目标；无合法目标时进入弃牌堆且不产生效果。

所有“本回合”“持续 N 回合”、飞行持续时间和临时属性默认按效果所属方的行动回合计数，不因对手行动回合经过而减少。

#### 关键词

- **飞行：** 高度变为 2，免疫所有攻击、可跨越地面障碍、只占用空中槽并失去被动涂色。
- **降落：** 书灵从高度 2 回到高度 1 时触发；若原坐标不可降落，使用最近合法地面格与坐标顺序平局规则。
- **精研：** 当单位移动到己方已涂色区域或在该区域释放法术时，触发卡牌定义的额外效果。
- **结晶化：** 格子变为半透明宝石质感并保持结晶瞬间的基础归属和地块状态。此后无法被任何地块效果影响；其上的书灵仍可正常移动、战斗并受到效果。
- **奥术护盾：** 每层抵挡下一次伤害事件，随后消耗。

### 3.15 敌人与奖励

#### 敌人意图

- HUD 始终显示敌人接下来 **3 个敌方行动回合**的计划。
- 每个意图至少包含卡牌/技能名称、目标或目标区域、预计墨水变化和召唤位置。
- 意图生成后必须锁定并严格执行，不得在玩家行动后重新选择更有利方案。
- 若到执行时目标失效，该意图对应卡牌直接牺牲，敌方获得 0.5 墨水并补牌。
- 随后敌方必须再选择一张手牌牺牲，再获得 0.5 墨水并补牌，然后结束操作阶段。
- 失效意图不得重新选择目标，也不得改打另一张可用卡。

#### 战斗奖励

- 胜利后展示三张卡，玩家选择一张加入当前局卡组。
- 三张奖励固定构成为：
  - 2 张来自玩家当前角色卡池。
  - 1 张来自本场敌人角色卡池。
- 敌人卡池可包含敌人专属“作弊”卡，玩家选中后可正常获得并加入卡组。
- 奖励生成使用稳定随机种子，并避免同一次三选一出现重复卡牌，除非卡池内容不足。

## 4. 游戏世界与美术方向

- **主题/背景：** 欲望强烈的人收到法师之塔邀请，在塔内通过卷轴战场、卡牌与书灵完成试炼。
- **叙事优先级：** 轻量背景叙事。角色机制、构筑差异和战场可读性优先于长篇剧情。
- **氛围：** 安静的法术图书馆、纸张、墨水、羽毛、结晶与逐步升高的战术压力。
- **美术风格：** 非像素 2D；纸质卷轴 + 墨迹边界 + 卡牌 UI + Q 版角色与书灵。
- **目标视口：** 1280 x 720。
- **书灵屏幕占比：** 约视口高度 8%，720p 下约 58 px 高。
- **色彩：**
  - 纸面：暖白、羊皮纸色。
  - 玩家：深蓝、墨黑。
  - 敌人：红色、紫罗兰。
  - 茜娜：浅蓝、纯白、羽毛与天鹅意象。
  - 菲涅：深蓝、棕色皮革、烫金与宝石结晶。
- **战场表现：** 方形卷轴内部放置轻微透视的六边形地图；视觉变形不改变逻辑坐标。
- **参考方向：** 《杀戮尖塔 2》的卡牌拖拽与目标箭头、《炉石传说》的单位互伤、《杀戮尖塔》的构筑信息密度，以及纸墨卷轴式区域控制。

## 7. UI / UX 设计

### 7.1 战斗 HUD

- **中央至中央左侧：** 方形卷轴与六边形战场是最大视觉区域，从画面中央向左延伸，并靠近左侧安全边缘；卷轴中心略偏左，不做严格几何居中。
- **战场占比：** 在 1280 x 720 下优先占据约 68%-74% 的可用宽度和大部分可用高度，同时保持方形卷轴完整可见。
- **右侧留白：** 右侧保留给敌人意图、战斗日志和辅助状态，不能覆盖卷轴的主要交互区域。
- **顶部：** 双方角色、墨水、涂色格数/百分比、完整轮与行动序号。
- **顶部时间轴：** 敌人未来三次意图，当前即将执行项最突出。
- **底部中央偏左：** 固定 5 张手牌与卷轴中心对齐，支持拖拽、悬浮与目标箭头。
- **底部左侧：** 当前角色技能、冷却和角色状态。
- **底部右侧：** 牌堆数、弃牌堆数、当前速度倍率。
- **右侧：** 可折叠战斗日志，记录操作、AI 决策、涂色、浸润、墨井和伤害。
- **卷轴内：** 书灵、护盾、飞行、结晶化、精研与目标高亮。

### 7.2 卡组构筑界面

- 支持拖拽改变卡牌顺序。
- 支持为每张卡设置 `0-9` 狂热数字优先级；`0` 最高，`9` 最低。
- 每张卡的墨滴下方显示可点击数字框，点击后直接选择 `0-9`。
- 同数字优先级的出牌顺序使用界面从前到后的卡牌排序。
- 每张卡右上角显示可点击的墨滴牺牲标志：红色为可牺牲，灰色为不可牺牲。
- 鼠标悬浮墨滴时显示可牺牲状态提示；点击只切换牺牲权限，不改变数字优先级。
- 墨水不足或没有合法可打牌时，狂热按 `9 -> 0` 从红色墨滴卡中牺牲；同数字按界面排序从后到前。
- 卡牌详情展示费用、类型、目标、完整规则、来源角色和关键词。
- 明确显示哪些最高优先级卡受保护，不会在狂热中被牺牲。

### 7.3 输入

- 左键：卡牌拖拽、目标选择、按钮确认、地图拖拽平移。
- 右键：取消拖拽、打开卡牌详情、关闭详情。
- 鼠标滚轮：缩放卷轴。
- Escape：暂停或返回上一级。
- Space：战场居中。
- 数字键 1-5：选择对应手牌；选中后仍需遵守目标确认。

### 7.4 暂停

暂停时冻结：

- 回合状态机。
- 书灵移动与动画时间。
- 狂热自动操作计时器。
- 特效和音频。

暂停菜单提供继续和返回主菜单。返回主菜单视为放弃当前战斗。

### 7.5 反馈

- 合法卡牌与目标使用清晰发光，不依赖颜色作为唯一信息。
- 红色/灰色牺牲墨滴除颜色外还必须有不同内部符号或描边，确保色觉差异下仍可识别。
- 无效召唤返回手牌并显示原因。
- 0.5 墨水通过半格图标与短暂 `+0.5` 浮字反馈。
- 涂色速率 2/3 的侧向格在书灵移动前短暂预览。
- 浸润使用从周边向中心收束的墨水动画。
- 墨井中立、玩家、敌人和转化中状态必须有不同外圈。
- 狂热开始和 70 阈值加速有明显但不遮挡战场的阶段提示。

## 11. Godot 4.x 技术方向

### 11.1 架构原则

- 战斗逻辑使用 ECS 和纯数据系统；场景树主要负责 UI、渲染、输入和编排。
- 所有规则必须可在无渲染环境中确定性模拟。
- 单一系统负责单一状态所有权，尤其是书灵逻辑坐标和格子归属。
- 物理或动画回调不得直接修改场景树，使用延迟调用或命令缓冲。
- 随机内容使用分离的种子流：地图生成、洗牌、敌人意图、奖励和卡牌随机效果互不干扰。

### 11.2 推荐数据结构

#### HexCell

### 11.3 推荐系统边界

- `HexCoordinateSystem`：odd-r、Axial、Cube 转换，邻居、距离、范围、镜像。
- `HexMapGenerationSystem`：1815 格创建、三分区墨井与出生锚点。
- `HexOccupancySystem`：地面/空中双槽占用、墨井地面占位和同高度排他。
- `TurnFlowSystem`：交替行动回合和单次操作状态机。
- `InkSystem`：半单位墨水、无自然回复、上限增长、墨井与书灵产出。
- `CardZoneSystem`：牌堆、手牌、弃牌、洗牌、立即补牌。
- `CardInteractionController`：拖拽阈值、目标箭头、取消、详情、墨滴牺牲权限、`0-9` 数字框与悬浮提示。
- `CardEffectSystem`：全部 51 张角色卡的数据驱动效果与八张重点卡的独立验收路径。
- `SpiritAISystem`：避战、扩张、好战评分。
- `SpiritMovementSystem`：路径逐步执行和边界处理。
- `PaintSystem`：涂色速率 0-3 与方向侧格。
- `CombatSystem`：同时互伤、护盾和死亡。
- `InfiltrationSystem`：基于快照的全图浸润。
- `InkWellSystem`：包围、转化和产出。
- `RoleSkillSystem`：起风、精读、冷却。
- `EnemyIntentSystem`：未来三回合生成、锁定与执行。
- `FrenzySystem`：自动卡牌优先级、牺牲和表现加速。
- `SettlementSystem`：>80%、第 90 阈值和结果。
- `BattleProjectionSystem`：将 ECS 状态投影到 HUD、地图和日志。

## 12. 待确认 / 待细化

以下项目不应由实现人员自行猜测。确认前必须数据化、可替换：

1. **边缘范围效果：** 7 格精读、5 x 5 或半径 2 范围只要部分越界，是整个操作无效，还是仅忽略越界格。
2. **未移动书灵的涂色速率 2/3：** 没有移动方向时是否仅涂脚下格。
3. **出生点：** 双方镜像三角阵型的精确出生锚点和三个相对坐标。
4. **Resource 墨井行为：** 需要明确“帮助包围/维持墨井”的目标半径、完成条件与被阻挡后的重新规划。
5. **技能冷却时序：** 使用技能后设置为 CD 1/2，并在下一次己方回合开始先减 1，是否符合期望的实际使用频率。
6. **护卫大天鹅区域重置：** 击退已确定为 3 格，但“将其占领区域重置为中立”具体是敌方书灵脚下格、移动路径，还是降落点半径 2。
7. **索引重排精研判定：** 判定传送前所在格、传送后落点，还是玩家选定时的目标格。
8. **矩形范围在六边形地图中的解释：** 卡牌文本中的 3 x 3 和 5 x 5 使用 odd-r 行列矩形，还是转换为半径相近的六边形范围。
9. **结晶化与特殊效果（已确定）：** 不能。ACE、死亡效果、区域净化及明确写明的重置或解除效果都不能改变已经结晶的地块。
10. **奖励卡池不足：** 玩家或敌人卡池不足以生成不重复三选一时的降级策略。
11. **正前方：** 冲锋白雀、莽撞突击等效果的朝向取最近一次移动方向、阵营默认前进方向，还是书灵独立朝向。
12. **资源书灵到期：** 持续时间归零时是直接移除、触发死亡，还是不触发死亡效果的退场。
